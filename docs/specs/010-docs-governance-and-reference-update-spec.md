## 1. Intent
Formalize when and how runtime-contract docs and locale navigation must be updated when behavior docs change.

## 2. In scope
- `docs/SUMMARY.md`
- `docs/README.md`
- locale hubs under `docs/i18n/*/`
- `docs/*-reference.md`

## 3. Requirements
- Define mandatory doc update criteria by touched subsystem.
- Require i18n follow-through when shared wording or IA changes.
- Include governance checklist references.

## 4. Acceptance criteria
- Any runtime-contract-relevant text change must list impacted locales and follow-through status.
